## 1. 시스템 개요

배달/운송 기사 채용 전 과정을 자동화하는 시스템.
모집 → 서류심사 → 전화스크리닝 → 확정/대기 → 온보딩 → 현장투입 → 이탈/재활용까지의 파이프라인을 구축한다.

### 핵심 원칙
- **셀레니움 사용 절대 금지** (외부 플랫폼 자동화 X)
- **Supabase가 메인 DB**, 구글 시트는 담당자용 대시보드
- **외부 노코드 도구(Make 등) 미사용** — Next.js API route에서 직접 처리
- 발신 메시징은 **SOLAPI 알림톡(우선) + SMS 폴백** — 기존 SMS-only에서 전환 중
- 수신은 **전용 폰 SMS Gateway** (카카오 채널 1:1 답장은 API 미제공으로 SMS 양방향 유지)
- 광고는 **메타 광고** (준비 완료)

### AS-IS vs TO-BE

| 단계 | 현재 (수동/SMS) | 자동화 후 |
|------|------------|-----------|
| 지원서 확인 | 시트 12개 따로 확인 | 관리자 대시보드 하나 |
| 지원 직후 안내 | 없음 | 알림톡 자동 발송 (템플릿 ①) |
| 무응답 리마인드 | 수동 재발송 | 24h 무응답 시 알림톡 자동 (템플릿 ②) |
| 서류심사 | 수동 확인 | 대시보드 필터 |
| 전화 스크리닝 | 유지 | 유지 |
| 확정/대기 메시지 | 복붙 SMS | 버튼 한 번 → 알림톡 (템플릿 ③/④) |
| 가이드 공유 | 매니저가 하나씩 | 확정 시 자동 (템플릿 ⑥) |
| 출근 전날 확인 | 전화 한 통씩 | 버튼 알림톡 자동 (템플릿 ⑤) |
| 슬롯별 인원 파악 | 수동 카운트 | 대시보드 위젯 |
| 대기자 재활용 | 시트 조건 수동 필터 | 필터 UI + N명 일괄 발송 |
| 이탈 관리 | 기록 없음 | churned_at/churn_reason 추적 |
| 외부 채널 유입 | 시트 산재 | /admin/manual-entry 수동 입력 폼 |
| 광고성 재호출 | 없음 | 채널 친구→친구톡 / 비친구+동의→SMS 분기 |
| 지원자 문자 확인 | 전용 폰 직접 확인 | 대시보드 대화 화면 |
| 문자 답장 | 전용 폰 직접 타이핑 | 대시보드에서 SOLAPI 발송 |
| 중복 지원 탐지 | 육안 | 전화번호 기준 자동 감지 |

## 3. 슬롯 구조

지점별 근무 슬롯 = **평일/주말 × 오전/오후 = 4개**. 각 슬롯당 **확정 2명 + 대기 1명** 편성.

| 슬롯 | 시간대 | 확정 정원 | 대기 정원 |
|--|--|--|--|
| 평일 오전 | 월~금 08:00 ~ 13:00 | 2 | 1 |
| 평일 오후 | 월~금 11:00 ~ 16:00 | 2 | 1 |
| 주말 오전 | 토~일 08:00 ~ 13:00 | 2 | 1 |
| 주말 오후 | 토~일 11:00 ~ 16:00 | 2 | 1 |

**지점당 정원**: 확정 8 + 대기 4 = **12명**. 12개 지점 총 정원 144명.

슬롯은 `applicants.confirmed_slot` 컬럼에 `평일오전 | 평일오후 | 주말오전 | 주말오후` 중 하나로 저장.

## 4. 모집 채널

### 채널 A — 메타 광고 (핵심 유입)
- 메타 광고 소재 → 클릭 시 지원 폼 랜딩
- URL: `https://recruitment-sooty.vercel.app/apply?source=meta&branch=광진자양`
- 상태: 광고 계정 준비 완료
- 주의: "특별 광고 카테고리(고용)" 설정 필요

### 채널 B — 카카오 오픈채팅 홍보봇 (보조)
- 오픈채팅방에 구인 메시지 + 폼 링크 자동 게시
- URL: `?source=kakao&branch=지점명`
- 상태: 코드 미작성

### 채널 C — 알바몬 / 알바천국 (수동)
- 수동 공고 등록, "홈페이지 접수"로 폼 URL 연결
- URL: `?source=albamon&branch=지점명`

### 채널 D — 외부 플랫폼 수동 입력 (신규 예정)
- 50+/당근 등 외부에서 유입된 지원자를 매니저가 `/admin/manual-entry` 폼으로 입력
- Supabase에 `source='manual'`로 통합 저장
- 전제: "모든 채널 → Supabase" 운영 규칙 합의

## 6. 데이터 저장

### 구조

```
지원자 폼 제출
    ↓
Supabase applicants 테이블 저장 (메인 DB)
    ↓
구글 시트에 동기화 (담당자용 대시보드)
    ↓
알림톡 ① 자동 발송
```

### Supabase 테이블

**applicants 주요 컬럼** (상세는 `supabase-schema.sql` + `supabase-migration-messages.sql` + `supabase-migration-alimtalk.sql` 참고)

| 컬럼 | 타입 | 용도 |
|--|--|--|
| id / phone / name / birth_date | 기본 식별 |
| branch / branch1 / branch2 | 지점 (URL/1지망/2지망) |
| work_hours | TEXT | 지원자 희망 시간대 (4종 콤마 join) |
| status | TEXT | '서류심사' / '확정' / '대기' / '온보딩' / '현장투입' / '이탈' |
| last_message_at / unread_count | 대화 트리거 자동 갱신 |
| **reminder_sent_at** | TIMESTAMPTZ | 리마인더 중복 방지 (신규) |
| **marketing_consent / _at** | BOOLEAN / TIMESTAMPTZ | 마케팅 수신동의 (신규) |
| **kakao_channel_friend** | BOOLEAN | 친구톡 발송 대상 판단 (신규) |
| **start_date** | DATE | 확정된 근무 시작일 (신규) |
| **confirmed_slot** | TEXT | 확정 슬롯 `평일오전` 등 (신규) |
| **confirmed_branch** | TEXT | 실제 배치 지점 (신규) |
| **current_branch** | TEXT | 현재 근무 중 지점 (null=비근무) (신규) |

| **attendance_response / _at** | 출근 전날 확인 응답 (신규) |

**messages 새 컬럼**
| 컬럼 | 용도 |
|--|--|
| **message_type** | 'sms' / 'alimtalk' / 'friendtalk' (신규) |
| **template_id** | 알림톡 템플릿 ID (신규) |

## 7. 자동화 흐름

### 전체 파이프라인

```
[유입]
 │  메타광고 / 알바몬 / 당근·50+ / 카카오 오픈채팅
 ▼
[1] /apply 폼 작성 (마케팅 수신동의 체크박스)
 │
 ▼
[2] Supabase 저장 + 슬랙 알림 + 구글 시트 동기화
 │  status = '서류심사'
 ▼
[3] 자동 알림톡 발송 — 템플릿 ① 서류접수 안내
 │  └─ 실패 시 SMS 자동 폴백
 ▼
[4a] 24h 무응답 → 리마인더 크론 → 템플릿 ② 발송
 │  reminder_sent_at 기록 (중복 방지)
 ▼
[5] 관리자: 대시보드에서 서류 검토 + 전화 스크리닝
 │
 ▼
[6] 관리자: [확정] / [대기] 버튼 클릭
 │  ├─ 확정 → 템플릿 ③ 확정공지 + 템플릿 ⑥ 가이드 자동 발송
 │  │         + confirmed_slot / start_date / confirmed_branch 기록
 │  └─ 대기 → 템플릿 ④ 대기안내 발송
 ▼
[7] 근무 전날 18:00 → 템플릿 ⑤ 출근확인 (버튼) 발송
 │  └─ 사용자 버튼 클릭 → 웹훅 → attendance_response 업데이트
 ▼
[8] 근무 시작 → 관리자 current_branch 입력
 │  status = '현장투입'
 ▼
[9] 이탈 시 [이탈 처리] → churned_at / churn_reason / current_branch=null
 │
 ▼
[10] 재모집 시 필터 UI → 대기자 일괄 알림톡/친구톡/SMS 분기 발송
```

### 문자 송수신 흐름

```
[발신 (outbound) — 알림톡 우선]
관리자/자동 트리거 → SOLAPI sendAlimtalk(pfId, templateId, variables)
                      ↓ (카카오 도달 실패 시 자동)
                    SMS 폴백 (disableSms: false)
                      ↓
                  messages 저장 (message_type, template_id 포함)

[수신 (inbound)]
지원자 문자 → 전용 폰 → SMS Gateway 앱 → Supabase messages
                                         ↓
                                   트리거: unread_count +1
```

### 트리거/크론 목록

| 트리거 | 실행 | 구현 방식 |
|--------|------|-----------|
| 폼 제출 | 저장 + 알림톡 ① 발송 + 슬랙 + 시트 | API route |
| 24h 무응답 | 알림톡 ② 리마인더 | Vercel Cron → API |
| [확정] 클릭 | 알림톡 ③ + ⑥ 발송 | 대시보드 → API |
| [대기] 클릭 | 알림톡 ④ 발송 | 대시보드 → API |
| 시작일 전날 18:00 | 알림톡 ⑤ 출근확인 | Vercel Cron → API |
| 출근확인 버튼 응답 | attendance_response 업데이트 | 웹훅 수신 route |
| 전용 폰 SMS 수신 | messages 저장 + 트리거 | SMS Gateway + DB trigger |
| 전용 폰 heartbeat | 5분마다 상태 ping | SMS Gateway WorkManager |
