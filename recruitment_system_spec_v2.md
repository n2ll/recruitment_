## 1. 시스템 개요

배달/운송 기사 채용 전 과정을 자동화하는 시스템.  
모집 → 서류심사 → 연락 → 전화스크리닝 → 확정/대기 → 온보딩 → 현장투입까지의 파이프라인을 구축한다.

### 핵심 원칙
- **셀레니움 사용 절대 금지** (알바몬/알바천국 등 외부 플랫폼 자동화 X)
- **모든 지원자 데이터는 하나의 커스텀 폼으로 수렴**시켜 마스터 구글 시트에 통합 저장
- 대시보드는 **마스터 구글 시트** (기존 10개 → 1개로 통합)
- 메시징은 **SOLAPI 문자 발송** (이미 등록 완료)
- 광고는 **메타 광고** (이미 준비 완료)

### AS-IS vs TO-BE

| 단계 | 현재 (수동) | 자동화 후 |
|------|------------|-----------|
| 지원서 확인 | 시트 12개 따로 로그인해서 확인 | 마스터 시트 하나만 보면 됨 |
| 설문/시작시간 문의 | 지원자마다 채팅 수동 발송 | 필터 통과 즉시 자동 발송 |
| 슬롯 확인 | 시트 탭 이동하며 수동 확인 | 슬롯 현황 시트에서 실시간 확인 |
| 전화 스크리닝 | 동일 (유지) | 동일 (유지) |
| 확정/대기 메시지 | 수동 발송 | 상태 변경 시 자동 발송 |
| 아이디 발송 안내 | 수동 발송 | 상태 변경 시 자동 발송 |
| 중복 지원 탐지 | 육안으로 발견 or 놓침 | 전화번호 기준 자동 감지 |

## 3. 모집 채널 (3개)

### 채널 A — 메타 광고 (핵심 유입)
- **역할**: 주력 지원자 유입 채널
- **구현**: 메타 광고 소재 → 클릭 시 커스텀 지원 폼으로 랜딩
- **URL 예시**: `https://[도메인]/apply?source=meta&branch=광진자양`
- **자동화 수준**: 완전 자동
- **상태**: 광고 계정 준비 완료
- **주의**: 채용 광고는 메타에서 "특별 광고 카테고리(고용)"으로 설정 필요. 미설정 시 타겟팅 제한 또는 광고 차단 가능

### 채널 B — 카카오 오픈채팅 홍보봇 (보조)
- **역할**: 오픈채팅방에 구인 메시지 자동 발송
- **구현**: 홍보봇 코드 작성 (제재 거의 없음, 팀 확인 완료)
- **동작**: 설정된 시간 간격으로 오픈채팅방에 구인 메시지 + 지원 폼 링크 자동 게시
- **URL 예시**: `https://[도메인]/apply?source=kakao&branch=광진자양`
- **자동화 수준**: 완전 자동

### 채널 C — 알바몬 / 알바천국 (수동)
- **역할**: 추가 유입 채널 (구직 의도 있는 사람 대상)
- **구현**: 수동으로 공고 등록, 접수 방법을 **"홈페이지 접수"**로 설정하여 커스텀 폼 URL 연결
- **URL 예시**: `https://[도메인]/apply?source=albamon&branch=광진자양`
- **자동화 수준**: 공고 등록만 수동, 이후 데이터 수집은 자동
- **리스크 대응**: 홈페이지 접수 정책 제한 시 → 플랫폼 자체 온라인 지원 병행 + 이메일 알림 파싱으로 반자동 처리

#### 알바몬/알바천국 자동화를 하지 않는 이유
- 두 플랫폼 모두 공고 등록용 공개 API 없음
- 알바몬은 프로그램 접근 탐지 시 자동 차단 시스템 운영 중
- 계정 정지 시 기존 지원자 데이터, 진행 중 공고, 유료 상품 전부 유실 → 사업 연속성 위협

## 4. 지원 폼 (커스텀 개발)

### 기술 스택
- **프레임워크**: Next.js (단일 페이지)
- **DB 연결**: Google Sheets API → 마스터 시트에 직접 행 추가

### 폼 필드

| 필드명 | 타입 | 필수 | 자동필터 |
|--------|------|------|----------|
| 성함 | text | ✅ | |
| 생년월일 6자리 | text | ✅ | |
| 휴대폰 번호 ('-' 제외) | text | ✅ | 중복 감지 |
| 거주지 (동단위) | text | ✅ | |
| 자기명의 차량 여부 | select (Y/N) | ✅ | ✅ 조건1 |
| 운전면허 종류 | select | ✅ | ✅ 조건2 |
| 차종 | text/select | ✅ | ✅ 조건3 |
| 희망 근무 시간대 | checkbox (중복 선택) | ✅ | |
| 자기소개 및 지원동기 | textarea | ✅ | |
| 배달 업무 관련 경력 | textarea | | |

#### 희망 근무 시간대 옵션
- 평일(월~금) 오전 타임 (08:00 ~ 13:00)
- 평일(월~금) 오후 타임 (11:00 ~ 16:00)
- 주말(토~일) 오전 타임 (08:00 ~ 13:00)
- 주말(토~일) 오후 타임 (11:00 ~ 16:00)

### 채널 추적
- URL 파라미터 `?source=` + `?branch=` 값을 자동 캡처하여 시트에 저장
- 채널별 유입 수, 필터 통과율, 최종 현장투입 비율 분석 가능

### 기존 구글폼과의 관계
- 현재 지점별 구글폼 12개로 운영 중 → 커스텀 폼 완성 시 URL 교체로 전환
- 커스텀 폼 완성 전까지 기존 구글폼 임시 유지

> 슬롯 수는 지점별/타임별로 언제든 직접 수정 가능. 확정 인원 추가 시 Make가 자동 차감.

### 자동 필터 로직 (Make)

새 행이 시트에 추가되면 Make가 트리거:

```
IF 자기명의 차량 여부 = Y        ← 조건1
   AND 운전면허 종류 ∈ 적합 목록  ← 조건2
   AND 차종 ∈ 적합 차종 목록      ← 조건3
THEN
  → filter_pass = Y
  → 진행상황 = "연락대기"
  → [메시지1] SOLAPI 자동 발송
ELSE
  → filter_pass = N
  → 진행상황 = "부적합"
  → 부적합 안내 문자 자동 발송
```

### 지점별 필터 뷰 세팅

구글 시트 상단 → 데이터 → 필터 뷰 → 새 필터 뷰 만들기  
`branch` 컬럼 기준으로 지점명 선택 후 저장.  
담당자마다 본인 지점 뷰를 저장해두면 열 때마다 자동 적용.

## 6. 실제 메시지 템플릿

### 메시지1 — 업무시작시간 + 설문링크 (필터 통과 즉시 자동 발송)

```
안녕하세요 {name}님. 기다려 주셔서 감사합니다.
옹고잉 매니저입니다!

1. 업무는 언제부터 가능하실까요?
2. 본인 명의로 업무 진행 및 정산에 문제 없으신지 말씀 부탁드립니다.
3. 하기 설문 링크를 통해 지원서 제출을 요청드립니다.

설문링크: https://[도메인]/apply?source={source}&branch={branch}
```

> 발송 트리거: 자동필터 통과 즉시  
> {name}, {source}, {branch} → Supabase/시트 데이터 자동 삽입  
> 담당자 할 일: 없음

### 메시지2 — 배민커넥트 앱 가입 + 아이디 요청 (확정 시 자동 발송)

```
{name}님, 함께하게 되어 반갑습니다!
아래 순서로 진행 부탁드립니다.

1. 배민 커넥트 앱 설치 후 가입
2. 앱 가입 시 안전보건교육 영상(2시간) 필수 시청
3. 가입 및 교육 수료 후
   마이페이지 > 내 정보에서 아이디 확인 후
   아이디 회신 부탁드립니다.

문의사항은 편하게 말씀주세요.

[가입 가이드 영상]
https://www.youtube.com/watch?v=bMM112zT7JY
```

> 발송 트리거: 진행상황 → "온보딩" 변경 시 Make 감지  
> 담당자 할 일: 시트에서 진행상황 셀 값 변경

### 부적합 안내 문자

```
[옹고잉] {name}님, 지원해주셔서 감사합니다.
아쉽게도 현재 모집 조건에 부합하지 않아 안내드립니다.
추후 조건이 맞는 기회가 생기면 다시 연락드리겠습니다.
```

## 7. 전체 프로세스 플로우

```
공고 등록 (알바몬/알바천국 수동, 단 1회)
메타 광고 집행 (자동)
카카오 홍보봇 (자동)
          ↓
커스텀 폼 (Next.js)
?source=meta&branch=광진자양
          ↓
마스터 구글 시트 행 자동 추가
          ↓
Make 트리거 — 자동 필터 3조건
    ↓ 통과                      ↓ 탈락
진행상황 = "연락대기"          진행상황 = "부적합"
[메시지1] 자동 발송            부적합 안내 문자 자동 발송
          ↓
담당자 2명 전화 스크리닝
(마스터 시트 한 곳에서 정보 확인)
슬롯 현황 시트에서 잔여 슬롯 실시간 확인
          ↓
       판단
  ↙          ↘
확정           대기
(슬롯 있음)   (슬롯 없음)
  ↓               ↓
진행상황          진행상황
= "온보딩"        = "대기"
슬롯 -1 자동      대기 안내 문자
[메시지2] 자동 발송
  ↓
아이디 회신 수집
(아이디 제출 폼 or 채팅)
  ↓
진행상황 = "현장투입"
투입 전 안내 문자 자동 발송
```

## 8. Make 자동화 트리거 목록

| 트리거 조건 | 자동 실행 |
|------------|-----------|
| 시트 새 행 추가 | 자동 필터 3조건 실행 |
| filter_pass = Y | 메시지1 발송 + 진행상황 = "연락대기" |
| filter_pass = N | 부적합 안내 문자 발송 |
| 진행상황 → "온보딩" | 메시지2 발송 + 슬롯 -1 자동 차감 |
| 진행상황 → "현장투입" | 투입 전 안내 문자 발송 |
| 전화번호 중복 감지 | 담당자 알림 (중복 지원 경고) |

---

## 9. 메시징 (SOLAPI)

- SOLAPI **이미 등록 완료** → 즉시 사용 가능
- **문자(SMS/LMS)로 시작**, 열람률 낮으면 알림톡으로 전환
- SOLAPI가 문자/알림톡 둘 다 지원 → 코드 한 줄 차이로 전환 가능

### 알림톡 전환 시 장점
- 건당 비용 절감 (문자 15~40원 → 알림톡 7~8원)
- 카카오톡 수신 → 스팸함 회피, 열람률 상승
- 버튼 기반 응답 가능 (참석/불참 클릭 → 웹훅 자동 처리)

## 11. 개발 일정

### Step 0 — 마스터 시트 수동 통합 (오늘, 1~2시간)
- 마스터 구글 시트 신규 생성
- 기존 12개 시트 데이터 복붙 + `branch` 컬럼 추가
- 슬롯 현황 시트 생성 (전 지점 기본값 3으로 세팅)
- 지점별 필터 뷰 12개 세팅
- **완료 시 효과**: 당장 내일부터 시트 12개 왔다갔다 없어짐

### Phase 1 — 지원 폼 + 시트 연동 (Day 1)
- Next.js 지원 폼 페이지 개발
- Google Sheets API 연결 (새 행 자동 추가)
- source + branch 파라미터 자동 캡처
- 중복 전화번호 감지 로직

### Phase 2 — Make 자동화 + 문자 연동 (Day 2)
- Make: 새 행 감지 → 자동 필터 3조건 실행
- Make: SOLAPI 메시지1/2 자동 발송
- Make: 슬롯 자동 차감 (진행상황 → "온보딩" 시)
- Make: 에러 알림 설정

### Phase 3 — 채널 + 테스트 (Day 3)
- 카카오 오픈채팅 홍보봇 코드 작성
- 메타 광고 랜딩 URL 세팅 (지점별 12개)
- 전체 파이프라인 E2E 테스트

**예상 소요: Step 0 오늘 + 개발 3일 → 즉시 운영 가능**

## 12. 파이프라인 단계별 자동화 수준

| 단계 | 설명 | 자동화 | 담당 |
|------|------|--------|------|
| 1. 모집 | 메타 광고 + 카카오봇 + 알바몬(수동) | 90% | 시스템 |
| 2. 서류심사 | Make 자동 필터 (3조건) | 100% | 시스템 |
| 3. 메시지1 발송 | 업무시작시간 + 설문링크 자동 발송 | 100% | 시스템 |
| 4. 전화 스크리닝 | 담당자 2명 직접 전화 | 0% | 담당자 |
| 5. 확정/대기 처리 | 시트 진행상황 변경 → 자동 트리거 | 90% | 시스템 |
| 6. 메시지2 발송 | 배민커넥트 가입 안내 자동 발송 | 100% | 시스템 |
| 7. 현장투입 | 투입 전 안내 문자 자동 발송 | 90% | 시스템 |
| 8. 사후관리 | 앱 지표 트래킹 → 이탈 시 순환 | 데이터 기반 | 담당자 |
