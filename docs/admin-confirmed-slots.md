# 확정 슬롯 탭 (`tab="confirmed-slots"`)

`status='확정'` 으로 배치된 인원의 정원(2명/슬롯) 충족 현황 매트릭스.

- 위치: [app/admin/page.tsx:1394-1487](../app/admin/page.tsx#L1394-L1487)
- 데이터 소스: 클라이언트 메모리 `applicants`

---

## 1. 매트릭스 셀 산식

행 = 활성 지점, 열 = 4슬롯.

```
confirmed(branch, slot) = data.filter(a =>
  a.status === "확정" &&
  a.confirmed_slot === slot &&
  a.confirmed_branch === branch
).length
```

각 셀에 `{n}/2` 표시 (정원 2명 기준).

---

## 2. 색상

| 조건 | 클래스 | 의미 |
|--|--|--|
| `n >= 2` | `cell-full` | 정원 충족 (초록) |
| `n === 1` | `cell-half` | 1명 (주황/노랑) |
| `n === 0` | `cell-zero` | 빈 슬롯 (회색) |

상단 범례 칩으로 색상 의미 설명.

---

## 3. 셀 클릭 → drill-down

- `slotCell={branch, slot}` 상태로 하단 테이블 노출
- 컬럼: 성함 / 연락처 / 상태 / 시작일 / 현재 근무지(`current_branch`)
- 행 클릭 → 지원자 목록 탭으로 점프 + 상세 패널 오픈

> 빈 슬롯이면 "지원자 목록에서 확정 슬롯/지점을 지정해주세요." 안내.

---

## 4. 사양과의 차이

[recruitment_system_spec_v2.md](../recruitment_system_spec_v2.md) 4번 섹션은:

- 확정 정원 2 + **대기 정원 1** = 슬롯당 3명
- `대기` status를 별도 색상(보라)으로 노출

→ 현재 코드는 **확정 2명만** 카운트. `대기` status 자체가 코드의 `ALL_STATUSES`에 없음. 사양서와 운영 실태가 어긋난 부분으로, 향후:
1. `대기` status를 도입해 사양에 맞추거나
2. 사양서를 현재 운영(확정만)에 맞게 갱신

---

## 5. 활용

- "어느 슬롯이 결원인가?" → 희망 슬롯 매트릭스 → drill-down → 후보 컨택
- 신규 지원자 확정 처리 후 매트릭스가 자동 갱신 (Realtime)
- 정원 초과(>2)도 셀에 그대로 표시됨 → 이중 배정 발견용
