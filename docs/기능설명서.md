# 1. 지원 폼 (`/apply`)

**용도**: 모든 채널(당근·페이스북·네이버 검색·기타)의 지원자가 모이는 단일 입구.

**입력 항목**: 이름·생년월일·전화·거주지(우편번호 검색)·자차 여부·면허·차종·1·2지망 지점·시간대·자기소개·시작가능일·본인명의 가능 여부·**지원경로(라디오 4개)**·마케팅 동의.

## 지원경로 라디오

`당근 / 페이스북 / 네이버 검색 / 해당없음` 4개 중 1개 선택.
- URL에 `?source=danggeun`이 있으면 자동으로 "당근" 선택
- 그 외엔 디폴트 "해당없음"
- DB에 영문 키(`danggeun`/`facebook`/`naver`/`direct`)로 저장. 화면 표시는 한글 매핑

## source='danggeun' 특별 처리

지원자가 "당근" 선택해서 제출하면:
1. `applicants` INSERT (source='danggeun')
2. `job_candidates` 자동 생성 (시스템 더미 공고 매핑 + agent_stage='exploration')
3. **매니저가 저장한 시작 멘트가 즉시 SMS 발송** (예: "안녕하세요! 당근에서 연락드린 정현강 매니저입니다")
4. 이후 지원자가 답장하면 router가 AI를 호출 → 자동 응대 시작

다른 source(direct/facebook/naver)는 기존 접수 안내 메시지만 나가고 자동 AI 응대는 X (매니저 수동).

## 자동 필터

다음 3개 모두 충족해야 "서류심사" 통과:
1. 자기 차량 **있음**
2. 면허 **1종 보통 / 2종 보통 / 1종 대형**
3. 본인 명의 **문제 없음**

→ 통과: 안내 발송 + 카카오 채널 추가 페이지 노출
→ 탈락: "부적합" 상태로 저장

## 2-1. AI 에이전트 — 🥕 당근마켓구인 (1차 목표)

**핵심 화면.** 당근(또는 라이브 source='danggeun')에서 들어온 후보를 AI가 자동 응대 + 매니저가 모니터링.

### 화면 구조
- **상단 툴바**: 후보 수 / **[⚠ 시작 멘트 설정 필요 or ✓]** / **[+ 새 당근 후보]** / **[⭐ 추천 받기]** / [새로고침]
- **좌측 목록**: source='danggeun' 후보 전체. 각 행에 이름 / stage badge / 전화 / 지점 / 마지막 대화 시각 / 안 읽음 배지
- **우측 대화창**: 선택된 후보의 대화. AI 메시지엔 🧠 판단 근거 표시. 4단계 progress + 체크리스트 + 매니저 직접 발송 입력

### 시작 멘트 (저장 1회로 끝)
- 툴바의 **[시작 멘트]** 버튼 클릭 → 모달에서 멘트 작성·저장
- DB(`prompt_examples.system_message/danggeun_start`)에 저장 — **모든 매니저 공유 + apply route도 같은 멘트 사용**
- 새 후보 등록 / source='danggeun' 폼 지원자 첫 발송에 자동 사용

### + 새 당근 후보 (매니저 수동 등록)
- 모달: 이름 / 전화 / 지점 / **어떤 공고로 모집?** (facts 항목 select)
- 선택한 facts의 title+body가 `applicants.introduction`에 저장 → AI 응답 컨텍스트에 자동 인용 (예: 강북미아 시급 물어보면 facts에 적힌 시급 그대로 답변)
- 등록과 동시에 시작 멘트가 **실제 SMS로 발송**됨

### ⭐ 추천 받기
- 모달: 공고 메모(자유 입력 또는 facts에서 자동 채움) → [추천 받기]
- `source='danggeun'`인 후보 풀에서 점수순(거리/차량/최신성)으로 정렬
- 각 행 옆 **[시작 멘트 발송]** 버튼 → 그 후보에게 즉시 실 SMS

### stage 진행 (4단계 progress)
대화창 헤더 아래에 가로 progress 표시: **탐색 → 스크리닝 → 온보딩 → 근무중**
- 지난 단계: 초록 ✓
- 현재 단계: 노란 강조
- 매니저 인계(paused)/중단(abort)은 우측 상단에 별도 플래그

### 체크리스트
스크리닝/온보딩 단계에 진입하면 progress 아래에 진행률 + 항목별 체크 표시:
- **스크리닝 8항목**: 시작일·자차·프로모션 안내·정산·공휴일·본인명의·업무시간·질문해소
- **온보딩 4항목**: 앱설치 안내·배민 아이디·차량번호·만남장소 안내

### 매니저 인계 (paused) — ⏸ 빨간 배너
AI가 답할 수 없는 사실 질문(facts에 없는 지점 시급 등)을 받으면 `transition: pause` → stage='paused' 전환.
1. **Slack 알림 자동 발송**: `⏸ 매니저 인계 필요 - 지점 / 지원자 / 사유` (`SLACK_WEBHOOK_URL` 있으면 무조건 발송)
2. 대화창 위에 빨간 배너 표시
3. 매니저가 우측 대화창에서 **직접 답장**
4. 답장 후 매니저가 **[▶ AI 응답 재개]** 버튼 클릭 (배너 안의 초록 버튼)
   - 확인 모달: "이 버튼을 누른 시점 이후 후보가 새로 보내는 메시지부터 AI가 응답합니다"
   - 클릭 시 stage가 직전 단계로 복귀 + paused_reason null
5. 이후 후보의 다음 답장부터 AI가 다시 인수

> 자동 재개 X — 매니저가 명시적으로 버튼 눌러야 함

### 온보딩 진입 후 AI off — 🛑 파란 배너
온보딩 단계는 정책상 AI 자동 응답 OFF. 매니저가 직접 응대.
- 단, AI는 stage 모듈을 호출하긴 함 → 배민 아이디/차량번호 패턴은 자동 감지하여 체크리스트 갱신
- 시스템 자동 발송(앱설치 안내·만남장소 안내)은 그대로 동작

### Realtime
다음 이벤트는 화면에서 자동 갱신 (새로고침 불필요):
- `applicants` (source='danggeun') INSERT/UPDATE/DELETE → 목록 즉시
- `messages` INSERT → 현재 보는 후보 대화창 즉시
- `job_candidates` * → stage badge 즉시

## 2-2. AI 에이전트 — 🧠 클로드 조련하기

**용도**: AI가 학습할 모든 정보를 한 곳에서. 세 카테고리로 관리.

### 세부 탭 (왼쪽부터)
1. **참고자료 (facts)** — AI가 사실로 인용할 운영 정보 (지점·시급·구인 상태·정책)
   - 예) 제목: `강북미아` / 본문: `시급 15,000~20,000원, 토일 08:00-16:00, 픽업 서울 강북구...`
   - 여기 없는 사실은 AI가 추측 금지 정책 → `pause`로 매니저 인계
2. **스크리닝 운영 문구 (screening)** — 스크리닝 단계 운영 항목/문구 (정산·프로모션 안내 등)
3. **대화 톤 (conversation)** — 매니저가 실제로 보낸 메시지 예시. AI가 이 톤·길이·말투를 모방

### 편집
- [+ 새 항목] → 카테고리 선택(통합 모드는 자동) + 제목 + 본문
- 행마다 편집/삭제/순서(▲▼) 변경
- **저장 즉시 60초 이내** AI 프롬프트에 반영 (캐시 TTL 60초)

### 기본 예시 가져오기
첫 사용 시 [기본 예시 가져오기] 버튼으로 conversation/screening 시드 데이터 일괄 INSERT. facts는 매니저가 직접 채움.

## 2-3. 운영

### 대시보드
출근 직후 5초 안에 "오늘 어디까지 와 있나" 파악. 6개 카드 + 지점별 표.

### 지원자 목록
모든 지원자 검색·필터·상태 변경 메인 화면.
- 위쪽: 지점·상태 드롭다운, 이름/전화 검색
- 표: 이름·연락처·지점·차량·면허·시작일·상태·**채널(한글 매핑)**·지원일
- 이름 클릭 → 컨택 대화창 / 행 클릭 → 상세 패널
- `1차 스크리닝 완료` 버튼 → 가이드 알림톡 자동 발송

### 배송원 컨택
지원자와 SMS 양방향 대화. AI 답변 초안 자동 생성.
- 안 읽음 있는 사람 항상 맨 위
- 메시지 옆에 🤖 reasoning 회색 박스 표시 (AI가 보낸 메시지만)
- AI가 정보 부족이라 판단하면 ⚠️ "AI 응대 불가" + Slack 알림

> 카카오 1:1 답장은 정책상 API 안 됨 — SMS 양방향만 가능.

## 2-4. 매트릭스 — 확정 슬롯

지점 × 4슬롯 매트릭스. 각 셀에 `n/capacity` (예: `1/2`, `0/3`).

- **정원은 지점관리 탭에서 편집** (이전엔 코드 상수 2였음)
- 색상: 초록(정원 충족) / 노랑(1명 이상) / 회색(빈 슬롯)
- 셀 클릭 → 배치된 인원 리스트

## 2-5. 관리 — 지점 관리

- 지점 추가 / 드래그 순서 변경 / 이름·활성·삭제
- **하단에 "슬롯 정원 매트릭스" 카드** — 지점 × 4슬롯에 정원 number input
- [변경사항 저장] 한 번에 모든 변경(이름·순서·활성·정원) 한꺼번에 PATCH

> 지점 이름 변경은 기존 지원자 데이터에 자동 반영 X. **폐점은 활성 토글 OFF** 권장.

## 2-6. 기타 — 접혀 있는 메뉴들

1차 목표 외 메뉴들. 사이드바 "기타 ▸" 클릭 시 펼침.

### 구인 에이전트
공고 단위 칸반. 당근마켓구인과는 별개 흐름 (공고 → 후보 추천 → 일괄 발송 → 칸반 모니터링).

### 희망 슬롯
지원자가 "희망한" 시간대 분포 매트릭스. 모집 우선순위 결정용.

### 추천 받기
공고 메모 → Claude가 픽업 주소 추출 → 거리·차량·최신성 점수로 후보 정렬 → 일괄 SMS.

> 당근 후보 추천은 🥕 당근마켓구인 탭의 ⭐ 추천 받기 모달 사용.

### 현장 매니저
만남장소 안내·확정 슬랙 알림에 사용할 매니저 정보. 이름/전화/담당 지점.

### 플레이그라운드
공고+가짜 지원자 시뮬레이션. 실 발송 X, DB 저장 X.

> 당근 후보 시뮬은 🧪 당근마켓구인 (연습용) 사용.

# 3. AI 응대 4단계 흐름

후보 1명 = `job_candidates` 1행. 인입 SMS마다 현재 단계의 AI가 호출됨.

## 단계별 책임

| 단계 | 시작 시점 | AI가 하는 일 | 시스템 자동 발송 (전이 시) |
|---|---|---|---|
| **탐색 (exploration)** | 시작 멘트 발송 직후 | 공고·근무조건 질문에 답. 지원의사 명확해지면 → 스크리닝 | 안내 묶음 1통 (정산·프로모션·업무시간) |
| **스크리닝 (screening)** | 탐색 advance | 미확인 항목(시작일·자차·본인명의·공휴일) 묶어 질의, 응답에서 yes/no 추출 | 가이드 1통 (배민 앱 설치 / 영상교육 / 아이디·차량번호 회신 요청) |
| **온보딩 (onboarding)** | 스크리닝 완료 | **AI 응답 OFF (매니저 직접)** — 단, 체크리스트는 패턴 감지로 자동 갱신 | 둘 다 수신 시: 만남장소 안내 1통 |
| **근무중 (active)** | 시작일 D-day cron | 자유 대화 응대 | 첫 출근 룰 안내 |

## 조건부 자동 처리 (스크리닝 진입 시)

- **자차 재확인** → 공고 자차 불필요면 자동 통과
- **공휴일 업무 가능?** → 슬롯에 "주말" 없으면 자동 통과

→ 평일 전용·차량 불필요 공고는 시작일·본인명의 2개만 묻고 끝.

## 외 단계

- **paused** — AI가 답할 수 없는 사실 질문 만나면 자동 전환. Slack 알림 + 매니저 인계. 매니저가 **[▶ AI 응답 재개]** 버튼 눌러야 풀림
- **abort** — 시작일 절대 불가, 자차 없음 확정, 본인명의 불가, 명시적 거절 시. status='부적합' 자동 처리

## 중복 발송 방지

transitions의 자동 발송(안내 묶음/GUIDE)이 직전 3개 outbound와 동일 본문이면 skip. AI advance 시그널에 reply_text 같이 와도 router가 무시(transitions가 책임).

# 4. 데이터 흐름 / 트리거

## 라이브 당근마켓구인 트리거 3종

### 🔵 등록 트리거 — 매니저가 시작
```
[+ 새 당근 후보] 모달 → POST /api/admin/agent/danggeun/start
  ├ applicants INSERT (source='danggeun', introduction = facts 본문)
  ├ ensureDanggeunSystemJob → job_candidates INSERT (stage='exploration')
  ├ 시작 멘트 SOLAPI sendSms
  └ messages INSERT (sent_by='danggeun-start')
```

또는

```
지원자가 /apply 폼 작성 (source='당근') → POST /api/apply
  ├ applicants INSERT (source='danggeun')
  ├ job_candidates 자동 생성 (시스템 더미 공고)
  └ 시작 멘트 SMS 발송 (sent_by='danggeun-start')
```

또는

```
[⭐ 추천 받기] → 후보 클릭 → POST /api/admin/messages/send
  └ 이미 존재하는 applicant에 시작 멘트만 발송 (sent_by='danggeun-recommend')
```

### 🟢 응답 트리거 — 후보 답장 (AI 자동 응답 핵심)
```
후보 답장 SMS → SMS Gateway 앱 (전용 폰) → POST /api/messages/inbound
  ├ phone으로 applicants 매칭
  ├ messages INSERT (inbound)
  ├ job_candidates 찾기 → agent_stage 확인
  │     paused/abort면 skip
  │     exploration/screening이면 router 호출
  │     onboarding이면 stage 호출(체크리스트만), reply 차단
  └ router.runAgentForCandidate()
        └ stage 모듈(Claude) → reply_text + transition
        └ SOLAPI sendSms + messages INSERT (sent_by='agent')
        └ advance면 transitions가 자동 발송 (안내 묶음 등)
```

### 🟡 매니저 직접 발송
```
대화창에서 매니저 입력 → POST /api/admin/messages/send
  └ SOLAPI sendSms + messages INSERT (sent_by='danggeun-manual')
```

## 연습용은 모든 발송이 simulate
- impersonate 라우트가 `applicant.source === 'danggeun_practice'`면 `simulate=true` 전달
- router/transitions 모두 SOLAPI 호출 skip, DB outbound 기록만
- Slack 알림도 skip

# 5. 매니저 일과 / 자주 헷갈리는 점

## 매니저 작업 빈도

| 빈도 | 작업 |
|---|---|
| **매일** | 🥕 당근마켓구인 — paused 후보 답변 + [▶ AI 재개] / 대시보드 확인 |
| **공고 등록 시** | 🥕 [+ 새 당근 후보] 또는 ⭐ 추천 받기로 일괄 발송 |
| **주 1회** | 🧠 클로드 조련하기 — facts에 신규 지점/시급 추가 |
| **주 1회** | 확정 슬롯 매트릭스 점검 |
| **변경 시** | 지점 관리 — 슬롯 정원 조정 |
| **분기** | 기타 그룹 메뉴들 점검 |

## 자주 헷갈리는 포인트

- **paused 후 답장만 보내고 [▶ AI 재개] 안 누르면 AI 영영 침묵** — 버튼 꼭 누르기
- **`facts`에 없는 사실은 AI가 절대 추측 X** → 새 지점 추가하면 facts에도 추가 필수
- **연습용 ↔ 라이브 시작 멘트는 한 곳(DB)에서 관리** — 연습에서 다듬으면 라이브에도 그대로 반영
- **연습용 등록은 진짜 데이터 격리** — source='danggeun_practice'로 분리. 라이브 목록에 안 보임
- **시작 멘트는 매니저 브라우저가 아니라 DB(prompt_examples)에 저장** — apply route(서버)도 같은 멘트 사용
- **apply 폼 + source='당근'이 들어와도 AI 응대가 막혀 있다면** → ⓛ system_message 마이그레이션 미실행 ② DanggeunView에서 시작 멘트 저장 안 함 ③ chk_jc_stage 마이그레이션 미실행 중 하나
- **인입 SMS는 전용 폰 폴링 주기만큼 지연** (1~2분 정상). 그보다 오래면 폰 상태 바 확인
