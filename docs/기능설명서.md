# 1. 지원 폼 (`/apply`)

**용도**: 모든 채널(메타·알바몬·오픈채팅 등)의 지원자가 모이는 단일 입구.

URL 파라미터로 출처·지점 자동 태깅: `?source=meta&branch=강북미아` → 1지망 자동 채워짐.

**입력 항목**: 이름·생년월일·전화·거주지(우편번호 검색)·자차 여부·면허·차종·1·2지망 지점·시간대·자기소개·시작가능일·본인명의 가능 여부·마케팅 동의.

**자동 필터 — 다음 3개 모두 충족해야 "서류심사" 통과**:
1. 자기 차량 **있음**
2. 면허 **1종 보통 / 2종 보통 / 1종 대형**
3. 본인 명의 **문제 없음**

→ 통과: 알림톡 ① 자동 발송 + 카카오 채널 추가 페이지 노출  
→ 탈락: "부적합" 상태로 들어감

> 거주지는 자동으로 위경도 변환되어 추천 기능에 사용됨.  
> 같은 전화번호로 재지원하면 "중복" 표시만 붙고 저장은 진행됨.

# 2. 관리자 대시보드 (`/admin`)

좌측 사이드바 10개 탭.

## 2-1. 대시보드

**용도**: 출근 직후 5초 안에 "오늘 어디까지 와 있나" 파악.

상단 6개 카드: 전체 / 오늘 지원 / 필터 통과 / 스크리닝 대기(밀리면 노란색) / 스크리닝 완료 / 확정.

아래는 지점별 표(전체 / 필터 통과 / 스크리닝 대기 / 통과율) — 어느 지점이 모집 잘 되고 어디 마른지 비교.

> 모든 숫자 실시간 갱신. "오늘"은 한국 시간 자정 기준.

---

## 2-2. 지원자 목록

**용도**: 모든 지원자 검색·필터·상태 변경의 메인 작업 화면.

### 사용법
- 위쪽: 지점·상태 드롭다운, 이름/전화 검색
- 표: 이름·연락처·지점·차량·면허·시작일·상태·채널·지원일
- **이름 클릭** → 컨택 대화창 열림
- **행 클릭** → 상세 패널 열림 (편집 + 읽기 전용 영역)
- **`1차 스크리닝 완료` 버튼** (필터 통과자 행에만 표시) → 가이드 알림톡 ⑥ 자동 발송 + 상태=스크리닝 완료

### 상세 패널 — 편집 영역
진행 상태 / 확정 슬롯 / 확정 지점 / 현재 근무 지점 / 시작일 / 이탈 사유  
**[저장] 안 누르고 닫으면 변경 사항 날아감**.  
상태를 "이탈"로 바꾸면 근무 지점·이탈 시각 자동 정리.

---

## 2-3. 희망 슬롯

**용도**: "어디에 모집을 더 해야 하나?" 결정용 매트릭스.

행=지점, 열=4슬롯(평일오전·오후·주말오전·오후). 각 셀=그 조건을 희망한 활성 지원자(서류심사·스크리닝 완료·확정) 수.

색깔:
- 회색(0명) — **위험, 시급 모집**
- 연 노랑(1~2명) — 부족
- 진 노랑(3명+) — 여유

셀 클릭 → 해당 지원자 목록 펼쳐짐.  
**1·2지망 모두 카운트**되므로 같은 사람이 여러 셀에 나타남.

---

## 2-4. 확정 슬롯

**용도**: 정원이 다 찼나? 어디가 비었나?

행=지점, 열=4슬롯, 각 셀에 `n/2` (정원 2명 기준).

색깔:
- 🟢 초록(2+) — 정원 충족
- 🟡 노랑(1) — 결원 위험
- ⚪ 회색(0) — 빈 자리

셀 클릭 → 누가 배치돼 있는지 표 확장. 이름 클릭 → 지원자 목록으로 점프.

> 정원 2명은 코드에 박혀 있음. 지점별 다른 정원이 필요하면 개발 요청.

---

## 2-5. 배송원 추천

**용도**: 결원·신규 모집 시 후보 풀에서 점수 정렬 후 일괄 SMS.

### 흐름
1. **공고 작성** — 메모로 던지면 Claude가 정형 공고로 다듬음 (빠진 항목은 `[?]`)
2. **차량 필요 / 불필요** 선택, 픽업 주소 (수동 보정 가능)
3. **추천 받기** → Claude가 픽업 주소 추출 + 거리 70 + 차량 20 + 최신성 10 = 100점 만점으로 상위 10명 정렬 (`+10명` 으로 50명까지 확장)
4. **체크 + 발송** — 미리보기 후 [N명에게 발송]

### 주의
- **회수 불가** — 미리보기 필수
- **희망 시간대 매칭 안 함** — 평일오전 공고에 주말만 희망한 사람도 후보로 들어옴, 매니저 검토 필요
- 점수 가중치 변경은 개발 요청
- 마케팅 미동의자 자동 제외 여부 운영팀 확인 필요

---

## 2-6. 배송원 컨택

**용도**: 지원자와 카톡처럼 SMS 양방향 대화. AI 답변 초안 자동 생성.

### 사용법
- 안 읽음 있는 사람 항상 맨 위(빨간 배지)
- 카드 클릭 → 카톡 스타일 대화창 오버레이
- Enter=발송, Shift+Enter=줄바꿈

### 🤖 AI 답변 초안
지원자 답장이 오면 자동으로 Claude가 초안 작성 → 입력창 위 노란 박스에 표시 (판단 근거 포함).

매니저 선택지: **그대로 보내기 / 수정 후 발송 / 무시**.

AI가 정보 부족이라 판단하면 ⚠️ "AI 응대 불가" + 슬랙 알림 → 매니저 직접 답변.

> 초안은 어디까지나 제안. 가격·시간·조건은 반드시 검수 후 발송.  
> 카카오 1:1 답장은 정책상 API 안 됨 — SMS 양방향만 가능.

---

## 2-7. 지점 관리

**용도**: `/apply` 폼 드롭다운, 모든 통계·필터의 지점 목록 관리.

- **지점 추가** / **드래그로 순서 변경** / 이름·활성·삭제
- **반드시 [변경사항 저장]** 눌러야 반영
- 우측에 "지원자 N명" 표시 (사용 중 표식)

> ⚠️ 지점 이름 변경은 기존 지원자 데이터에 자동 반영 안 됨. **폐점은 활성 토글 OFF**로 처리.

---

## 2-8. 현장 매니저

**용도**: 만남장소 안내·확정 슬랙 알림·라인 담당 정보의 출처.

매니저 1명 = 1행. 입력 필드: **이름 / 전화번호 / 담당 지점**. 활성 토글로 on/off.

여기서 등록한 매니저를 [구인 에이전트] 의 공고 생성 시 "현장 매니저"로 배정 → 그 공고에 묶인 지원자들의 만남장소 안내 메시지에 자동 사용됨.

> 행별 [저장] 버튼 누르기 전엔 변경사항 미반영.  
> 삭제는 즉시 반영(되돌릴 수 없음).

## 2-9. 구인 에이전트

**용도**: AI가 SMS로 지원자 응대를 자동으로 끌고 가는 핵심 화면. 공고 단위로 후보를 묶어 칸반(단계별)으로 진행 관리.

### 화면 구성
- **상단**: 활성 공고 칩 + [+ 새 공고]
- **선택된 공고**: 칸반(탐색 / 스크리닝 / 온보딩 / 근무중 / 일시정지 / 부적합) + 후보자 표
- **우측 슬라이드 패널** (후보자 클릭 시): 정보 / 체크리스트 / 대화 / 매니저 액션(일시정지·재개·부적합 처리)

### 새 공고 만들기
1. 메모만 던지면 Claude가 본문 다듬음
2. 메타(지점·슬롯·시작일·차량 필요·픽업주소·**현장 매니저**·정원) 검토
3. 후보 추천 받기 → 체크 → [공고 저장 + 발송]
4. 자동으로 후보들이 SMS 받고 **탐색** 단계로 진입

### AI가 대신 처리하는 것
- 지원자 응답을 단계별 프롬프트로 처리 → 답변 + 체크리스트 갱신 + 단계 전이
- 단계 전이 시 시스템 자동 발송 (안내 묶음 / 가이드 / 만남장소 / 첫출근 룰)
- 확정 시 슬랙 알림 (라인명 / 지원자 / 매니저)
- 만남장소 정보(일시·위치·매니저)는 공고 메타 + 매니저 데이터에서 자동 조립

### 매니저가 개입하는 시점
- AI가 "pause" 한 경우 (정책 질문 등) → 패널에서 직접 답변
- 부적합 처리 / 강제 일시정지 / 단계 수동 변경
- 공고 마감

> 자세한 4단계 흐름은 § 3 참조.

## 2-10. 플레이그라운드

**용도**: AI 응대를 시뮬레이션하는 안전한 테스트장. **실제 발송·DB 저장 없음**.

### 흐름
1. **좌측**: 가짜 공고(자동생성 가능) + 가짜 지원자 정보 + 단계 선택(탐색/스크리닝/온보딩/근무중)
2. **우측**: 채팅 시뮬 — 매니저가 "지원자에 빙의"해서 답장 입력
3. AI가 어떻게 답하는지, 단계 전이가 일어나는지, 자동 발송 메시지 미리보기까지 확인 가능

### 어디에 쓰나
- 새 공고 작성 전 응대 톤 점검
- 까다로운 시나리오(거절·재촉·정책 질문) 테스트
- 프롬프트 변경 후 회귀 점검

> 새로고침하면 시뮬 사라짐. 진짜 발송은 [구인 에이전트] 탭에서.

# 3. 구인 에이전트 4단계 흐름

후보 1명 = `job_candidates` 1행. 인입 SMS마다 현재 단계의 AI가 호출됨.

## 단계별 책임

| 단계 | 시작 시점 | AI가 하는 일 | 시스템 자동 발송 (전이 시) |
|---|---|---|---|
| **탐색 (exploration)** | 매니저가 공고 발송 직후 | 공고·근무조건 질문에 답만 함. 지원의사 명확해지면 → 스크리닝 | 안내 묶음 1통 (정산·프로모션·업무시간) |
| **스크리닝 (screening)** | 탐색에서 advance | 미확인 확인질문(시작일·자차·본인명의·공휴일)을 한 메시지에 묶어 질의. 응답에서 yes/no 추출 | 가이드 1통 (배민 앱 설치 / 영상교육 / 아이디·차량번호 회신 요청) |
| **온보딩 (onboarding)** | 스크리닝 완료(체크리스트 모두 true) | 배민 아이디·차량번호 수집. 둘 중 하나만 받으면 다른 것도 요청 | 둘 다 수신 시: 만남장소 안내 1통 (일시·위치·현장 매니저) |
| **근무중 (active)** | 시작일 D-day cron 자동 | 자유 대화 응대 | 첫 출근 룰 안내 (출근 시간·식사·배차) |

## 조건부 자동 처리

스크리닝 진입 시 공고 조건에 따라 자동 처리되는 항목:
- **자차 재확인** → 공고가 자차 불필요면 자동 통과 (안 묻음)
- **공휴일 업무 가능?** → 슬롯에 "주말" 없으면 자동 통과

→ 결과적으로 평일 전용·차량 불필요 공고는 시작일·본인명의 2개만 묻고 끝.

## 외 단계

- **일시정지 (paused)** — AI가 정책/예외 인지 → 매니저 직접 응대 대기. 매니저가 답한 후 단계를 다시 변경하면 AI 재개.
- **부적합 (abort)** — 시작일 절대 불가, 자차 없음 확정, 본인명의 불가, 명시적 거절 시. `applicants.status='부적합'` 자동 처리.

## 슬랙 알림

`SLACK_NOTIFICATIONS_ENABLED=1` + Webhook URL 설정 시:
- **확정**(스크리닝 → 온보딩) 시: "라인 / 지원자 / 지점 / 매니저"
- **AI 응대 불가**(pause) 시: "지원자 / 부족한 정보"

# 4. 매니저 일과 / 자주 헷갈리는 점

## 매니저 작업 빈도

| 빈도 | 작업 |
|---|---|
| **매일** | 대시보드 → 컨택 탭 안 읽음 처리 → 구인 에이전트 paused 후보 답변 |
| **수시** | 결원 발생 시 추천 탭으로 일괄 발송 |
| **공고 단위** | 구인 에이전트 [+ 새 공고] → 추천 받기 → 발송. 이후엔 칸반만 모니터링 |
| **주 1회** | 희망/확정 슬롯 매트릭스 점검 → 광고·모집 우선순위 |
| **분기** | 지점·현장 매니저 탭 정리 |

## 자주 헷갈리는 포인트

- **상세 패널 변경은 [저장] 안 누르면 사라짐**.
- **알림톡 → SMS 폴백은 정상**. 템플릿 심사 통과 전엔 SMS로 나감.
- **추천 탭 일괄 발송은 회수 불가** — 미리보기 필수.
