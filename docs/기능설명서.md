# 1. 지원 폼 (`/apply`)

**용도**: 모든 채널(당근·페이스북·네이버 검색·기타)의 지원자가 모이는 단일 입구.

**섹션 6개**:
1. 기본 정보 — 이름·생년월일·전화·거주지(우편번호 검색)
2. 차량 — 자차 여부·면허·차종
3. 희망 근무 지점 — 1지망(필수)·2지망(선택)
4. 희망 근무 시간대 — 평일오전/평일오후/주말오전/주말오후 **복수 선택**
5. 경력 — 선택 입력
6. 추가 확인 사항 — 시작 가능일·본인 명의·지원 경로·마케팅 동의

> **자기소개 섹션은 제거**됨 (응답률 개선). DB 컬럼은 유지 — 매니저가 admin에서 메모 가능.

## 지원경로 라디오

`당근 / 페이스북 / 네이버 검색 / 해당없음` 중 1개. URL에 `?source=danggeun`이 있으면 자동 "당근".
DB에 영문 키(`danggeun`/`facebook`/`naver`/`direct`) 저장. 화면 표시는 한글 매핑.

## source='danggeun' 특별 처리

당근 채널로 들어오면:
1. `applicants` INSERT (source='danggeun')
2. `job_candidates` 자동 생성 (시스템 더미 공고 매핑 + agent_stage='**screening**')
3. screening 진입 시 자동 true 항목 미리 채움 (안내 묶음 + 평일 슬롯이면 공휴일 자동 통과)
4. **저장된 시작 멘트가 즉시 SMS 발송** (`prompt_examples.system_message/danggeun_start`)
5. 이후 지원자 답장은 router → screening 모듈 → AI 자동 응답

> 다른 source(direct/facebook/naver)는 기본 접수 안내 메시지만 나가고 자동 AI 응대는 X (매니저 수동).

## 자동 필터

3개 모두 충족해야 통과:
1. 자기 차량 **있음**
2. 면허 **1종 보통 / 2종 보통 / 1종 대형**
3. 본인 명의 **문제 없음**

→ 통과: `status='스크리닝'` + 안내 발송 + 카카오 채널 추가 페이지 노출
→ 탈락: `status='부적합'`

## 지점 on/off

`/api/branches`는 active=true 지점만 반환 → **/apply 폼에는 비활성 지점이 안 보임**.
admin 화면(목록·대시보드·매트릭스·모달)은 모든 지점 그대로 노출 (매니저 작업 편의).

## 2-1. 당근마켓구인 (1차 목표)

**핵심 화면.** 당근 + apply(source=danggeun) 후보를 AI가 자동 응대 + 매니저 모니터링.

### 화면 구조
- **상단 툴바**: 후보 수 / **[+ 새 당근 후보]** / **[⭐ 추천 받기]** / [새로고침]
- **좌측 목록**: source='danggeun' 후보. 이름 / stage 배지 / 전화 / 지점 / 마지막 대화 시각 / 안 읽음
- **우측 대화창**: 선택 후보의 대화. AI 메시지엔 🧠 판단 근거 표시
  - 상단 **[⏸ AI 일시정지]** / **[새로고침]**
  - 단계 progress (클릭 가능) + 체크리스트 + 매니저 직접 발송 입력

### 시작 멘트 편집
**[🧠 클로드 조련하기 → 자동 발송 메시지 → `danggeun_start`]** 에서 편집. 한 곳에서 모든 매니저 공유.
`{{이름}}` / `{{지점}}` / `{{시간대}}` placeholder 지원.

### + 새 당근 후보 (매니저 수동 등록)
- 모달: 이름 / 전화 / 지점 / **어떤 공고로 모집?** (facts 항목 select 또는 자유입력)
- 선택 내용이 `applicants.introduction`에 저장 → AI 응답 컨텍스트에 자동 인용
- 등록과 동시에 시작 멘트가 **실제 SMS로 발송**됨

### ⭐ 추천 받기
- 모달: 공고 메모 → `source='danggeun'` 후보 풀에서 점수(거리/차량/최신성)순 정렬
- 각 행 옆 **[시작 멘트 발송]** 버튼

### 단계 progress (3단계 표시 — 클릭 가능)

```
스크리닝 → 온보딩 → 확정
```

> 탐색(exploration)은 base 능력으로 상시 깔려 있어 progress에 표시 X. 프로세스는 스크리닝부터.

- 지난 단계: 초록 ✓
- 현재 단계: 노란 강조
- **다른 단계 노드를 클릭하면** 수동으로 그 단계로 이동 (이전 단계 체크리스트 자동 채움)
  - **스크리닝/온보딩** 클릭: `applyTransition(advance)` 그대로 태움 → 자동 발송·status 업데이트 포함
  - **확정** 클릭: 자동 발송 없이 stage + 체크리스트 + `applicants.status='확정'`만 갱신

### ⏸ AI 일시정지 / ▶ 재개
- 우상단 **[⏸ AI 일시정지]** 클릭 → stage='paused' + Slack '매니저 인계 필요' 알림
- 빨간 배너에 **[▶ AI 응답 재개]** 버튼 → 직전 stage로 복귀 (`paused_from_stage` meta 기록)
- 자동 paused (정책 질문·부분 가능일 등)와 동일한 메커니즘 — 매니저가 직접 응대 후 재개

### 체크리스트
- **스크리닝 8항목**: 시작일·자차·프로모션 안내·정산·공휴일·본인명의·업무시간·질문해소
  - 시작 멘트에 안내 묶음 포함 → 3·4·7번은 등록 시 자동 true
  - 평일 슬롯이면 공휴일(5)도 자동 true, 자차 불필요 공고면 자차(2)도 자동 true
- **온보딩 4항목**: 앱설치 안내·배민 아이디·차량번호·만남장소 안내
  - 앱설치(1): 온보딩 진입 시 자동 발송 → 자동 true
  - 만남장소(4): **이제 자동 발송 안 함** — 매니저가 직접

### 자동 pause 케이스 (모두 슬랙 알림)
1. AI가 답할 수 없는 사실 질문 (facts에 없는 시급 등) → "정책 질문" pause
2. **부분 가능일 답변** — work_hours에 '평일' 포함 시 월~금 중 하루라도 불가 / '주말' 포함 시 토·일 중 하루라도 불가
3. 매니저 수동 일시정지

### Realtime 자동 갱신
- `applicants` (source='danggeun') 변경 → 목록·뱃지 즉시
- `messages` INSERT → 대화창 즉시 + (열린 후보의) 단계/체크리스트 silent fetch
- `job_candidates` 변경 → stage 배지·체크리스트 즉시

### 응답 텀 (1분)
인입 SMS 시각 + 60초 후에 AI 답장 발송.
- "바로 답장" 위화감 제거
- sleep 중에 같은 후보가 추가 메시지 보내면 **더 늦은 핸들러가 모두 묶어 1회 답변** (coalesce)
- simulate(연습 빙의) / draft 미리보기는 텀 없음

### 마무리 멘트 + 24h 마감 안내
온보딩 진입 시 발송되는 가이드 본문 끝에 자동 부착:
> `⏰ {시각}까지 아이디와 차량번호를 반드시 회신해주세요.`

`{시각}` = 발송시각 + 24h을 **정각으로 반올림** (30분 ≥ → 다음 정각, 미만 → 이전 정각).
예) 17:31 발송 → "내일 18:00까지" / 17:14 발송 → "내일 17:00까지"

### sent_by 라벨 매핑 (대화창 표시)
| 원시 sent_by | 화면 |
|---|---|
| `system-*` (system-auto, system-onboarding-reminder, system-bulk 등) | 자동 발송 메시지 |
| `danggeun-manual` | 매니저 |
| `agent` / `agent-practice` / 그 외 `danggeun-*` (start/practice-start/recommend) | AI 에이전트 |

## 2-2. 클로드 조련하기

**용도**: AI가 학습할 정보를 한 곳에서. 표시 카테고리 2개.

### 탭 (기본 노출)
1. **운영 정보 (facts)** — AI가 사실로 인용 가능한 운영 정보 (지점·시급·구인 상태·정책)
   - 예) 제목: `강북미아` / 본문: `시급 15,000~20,000원, 토일 08:00-16:00, 픽업 서울 강북구...`
   - 여기 없는 사실은 AI가 추측 금지 → `pause`로 매니저 인계
2. **자동 발송 메시지 (system_message)** — 시스템이 자동 발송하는 고정 문구
   - 키 목록 (제목 = 키. **변경 금지**):
     - `danggeun_start` — 당근 후보 첫 발송
     - `apply_received` — apply 폼 접수 안내
     - `screening_announce` — 스크리닝 진입 시 안내 묶음
     - `onboarding_guide` — 앱설치·교육 안내 (마감시각 자동 부착)
     - `onboarding_reminder` — 24h 미회신 시 리마인더
     - `first_day_rules` — 근무 시작 안내
   - 본문에 `{{이름}}`, `{{지점}}`, `{{시간대}}`, `{{마감시각}}` placeholder 사용 가능

### 비노출 카테고리 (백엔드는 그대로 로드)
- **대화 톤 (conversation)** — UI에서 안 보이지만 `buildToneGuide`가 DB에서 그대로 로드해 AI 프롬프트에 주입
- **스크리닝 운영 문구 (screening)** — deprecated, 미사용

### 편집
- 저장 즉시 **60초 이내** AI 프롬프트에 반영 (캐시 TTL)
- [기본값 채우기] 버튼 — 시드 INSERT (멱등)

## 2-3. 운영

### 대시보드
출근 직후 "오늘 어디까지 와 있나" 파악. 6개 카드(전체/오늘/필터 통과/스크리닝/온보딩/확정) + 지점별 표.

### 지원자 목록
모든 지원자 검색·필터·관리 메인 화면.
- 위쪽: **[+ 지원자 추가]** 버튼 / 지점·상태 드롭다운 / 이름·전화 검색
- 표: 이름·연락처·지점·차량·면허·시작일·상태·**채널(한글)**·지원일
- 이름 클릭 → 컨택 대화창 / 행 클릭 → 상세 패널
- 상세 패널에서 인라인 편집(상태·확정 슬롯·확정 지점·시작일·이탈 사유) + **[✏️ 전체 편집]** 버튼 (모달, 모든 컬럼)
- 상태 5종: **스크리닝 / 온보딩 / 확정 / 이탈 / 부적합** (당근 단계와 통일)
- 확정 슬롯 — **복수 선택**(콤마 join). 매트릭스에 모두 카운트됨

### 수기 지원자 등록 (모달)
**[+ 지원자 추가]** 클릭 → 6섹션 폼:
1. 기본 정보 (이름·전화·생년월일·거주지·지원경로)
2. 차량 (보유·면허·차종·본인명의)
3. 희망 (1·2지망·시간대·시작가능일)
4. 상태·확정 (status·필터·확정 지점/슬롯·현재 근무지점·시작일·이탈 사유)
5. 메모·자기소개·경력
6. 마케팅 동의·카카오 친구

- 필수: 이름·전화·1지망
- 기본값: source='manual' / status='스크리닝'
- location 입력 시 자동 지오코딩

### 배송원 컨택
지원자와 SMS 양방향 대화. AI 답변 초안 자동 생성.
- 안 읽음 있는 사람 항상 맨 위
- AI가 정보 부족이라 판단 → ⚠️ "AI 응대 불가" + Slack 알림

> 카카오 1:1 답장은 정책상 API 안 됨 — SMS 양방향만.

## 2-4. 매트릭스 — 확정 슬롯

지점 × 4슬롯 매트릭스. 각 셀 `n/capacity`.
- 정원은 지점관리 탭에서 편집
- 색상: 초록(정원 충족) / 노랑(1명 이상) / 회색(빈 슬롯)
- 셀 클릭 → 해당 슬롯·지점의 인원 리스트
- **확정 슬롯이 복수**인 후보는 해당 슬롯 모두에 카운트됨

---

## 2-5. 관리 — 지점 관리

- 지점 추가 / 드래그 순서 변경 / 이름·**활성 토글**·삭제
- 하단 "슬롯 정원 매트릭스" — 지점 × 4슬롯 number input
  - 셀 클릭 시 자동 전체 선택 → 기존 값이 한 번에 교체됨
- [변경사항 저장] 한 번에 PATCH

> **활성 토글 OFF의 의미**: `/api/branches`(공개 폼 전용)에서만 제외 → /apply에 안 보임. admin 모든 화면은 그대로 표시.

# 3. AI 응대 단계

후보 1명 = `job_candidates` 1행. 인입 SMS마다 현재 단계 모듈이 호출됨.

## 단계 흐름

```
스크리닝 → 온보딩 → 확정(active)
   │         │
   └── paused (자동/수동) ←─ 매니저 [▶ 재개]
   │         │
   └── abort (부적합)
```

> 탐색(exploration)은 모듈로 존재하나 1차 목표 흐름에선 사용 X — 등록 시점에 바로 스크리닝 진입.

## 단계별 책임

| 단계 | 진입 시점 | AI가 하는 일 | 시스템 자동 발송 (진입 시) | applicants.status |
|---|---|---|---|---|
| **스크리닝** | 당근 후보 등록 시 | 8항목 체크리스트 진행. 미확인 항목 묶어 질의 | 시작 멘트(danggeun_start)에 안내 묶음 포함 | `스크리닝` |
| **온보딩** | 스크리닝 8/8 완료 | 배민 아이디·차량번호 수집. 둘 다 받으면 마무리 멘트 1회 | 앱설치/교육 가이드 (`onboarding_guide`) + 24h 마감 안내 자동 부착 | `온보딩` |
| **확정 (active)** | 매니저 수동 / D-day cron | (자유 대화) | 매니저 수동 진입 시 X. cron 진입 시 `first_day_rules` | `확정` |

## 자동 처리 규칙

- **스크리닝 진입 시 자동 true (등록 routes에서 미리 채움)**:
  - 시작 멘트에 안내 묶음 포함 → 프로모션 안내(3)·정산 안내(4)·업무시간(7)
  - work_hours에 '주말' 없음(평일만) → 공휴일 업무 확인(5)
  - 공고 vehicle_required=false → 자차 재확인(2)
- **온보딩 진입 시 자동 true**: 앱설치 안내(1)
- **온보딩 → 매니저 인계 이벤트**:
  - 배민 아이디 + 차량번호 둘 다 수신 시 슬랙 *'온보딩 준비 완료'* 알림 → 매니저가 만남장소·확정 처리
  - 24h 미회신 시 cron이 리마인더 SMS 1회 → 추가 3h 미회신 시 슬랙 *'매니저 전화 인계 필요'*

## paused / abort

- **paused**: 자동(사실 부족·부분 가능일·정책 질문) 또는 매니저 수동. Slack 알림 자동. 매니저 [▶ AI 재개] 누를 때까지 AI 침묵
- **abort**: 시작일 절대 불가 / 자차 없음 / 본인명의 불가 등 자격 미달. `status='부적합'` 자동

## 중복 발송 방지

- 자동 발송(안내 묶음/GUIDE/마무리) 직전 3개 outbound와 동일 본문이면 skip
- AI advance 시그널에 reply_text 같이 와도 router가 무시 (transitions가 책임)
- 온보딩 마무리 멘트: 둘 다 처음 채워진 턴에만 1회 발송 (이미 둘 다 있는 상태에서 또 인입 시 reply 강제 비움)

     (당근 흐름 외 매니저 전용 입력)
```

## 🟢 응답 트리거 (AI 자동 응답)

```
후보 답장 SMS → SMS Gateway → POST /api/messages/inbound
  ├ phone으로 applicants 매칭
  ├ messages INSERT (inbound)
  ├ job_candidates.agent_stage 확인
  │     paused/abort → skip (매니저 직접 응대)
  │     screening/onboarding/active → router 호출
  └ router.runAgentForCandidate({ received_at })
        ├ 1분 sleep (시작시각 + 60s 목표)
        ├ coalesce 체크 — 내 인입 이후 더 새로운 인입 있으면 양보
        ├ stage 모듈(Claude) → reply_text + transition
        ├ SOLAPI sendSms + messages INSERT (sent_by='agent')
        └ transitions.applyTransition — 자동 발송 + status 갱신 + Slack
```

```
대화창 입력 → POST /api/admin/messages/send (또는 당근 탭 매니저 발송)

|---|---|

# 5. 자동 발송·리마인더 (cron)

`vercel.json`의 `crons`로 등록.

## `/api/admin/cron/onboarding-reminder` (매시간 `0 * * * *`)

온보딩 단계 후보를 매시간 점검해 두 단계로 동작:

### 단계 A — 리마인더 SMS
조건: `agent_stage='onboarding'` AND `meta.onboarding_entered_at < now - 24h` AND
`meta.onboarding_reminder_sent_at IS NULL` AND (아이디 또는 차량번호 미수신)
→ `system_message.onboarding_reminder` 본문으로 SMS 1회 발송
→ `meta.onboarding_reminder_sent_at` 기록

### 단계 B — 매니저 전화 인계 슬랙
조건: `meta.onboarding_reminder_sent_at < now - 3h` AND
`meta.manager_handoff_alerted_at IS NULL` AND (여전히 둘 다 미수신)
→ Slack 🚨 *"📞 매니저 전화 인계 필요 — 온보딩 미회신"* 1회
→ `meta.manager_handoff_alerted_at` 기록

> 둘 다 수신된 후보는 어느 단계도 발동 안 함. 연습용은 슬랙 skip + 메타만 갱신.

# 6. 매니저 일과 / 자주 헷갈리는 점

## 작업 빈도

| 빈도 | 작업 |
|---|---|
| **수시 (실시간)** | 🥕 당근마켓구인 — paused 후보 답변 + [▶ AI 재개], ⏸ 수동 일시정지 |
| **공고 등록 시** | 🥕 [+ 새 당근 후보] 또는 ⭐ 추천 받기 |
| **매일** | 대시보드 / 지원자 목록 확인 |
| **주 1회** | 🧠 클로드 조련하기 — 운영 정보(facts) 갱신 |
| **변경 시** | 지점 관리 — 슬롯 정원, 활성 토글 |
| **수기 입력** | 지원자 목록 [+ 지원자 추가] 또는 [✏️ 전체 편집] |

## 자주 헷갈리는 포인트

- **paused 후 답장만 보내고 [▶ AI 재개] 안 누르면 AI 영영 침묵** — 버튼 꼭 누르기
- **`facts`(운영 정보)에 없는 사실은 AI가 절대 추측 X** → 새 지점 추가 시 facts에도 추가
- **응답 텀 1분** — AI 답장이 60초 후에 도착하는 게 정상. 빨리 보내고 싶으면 매니저 직접 발송
- **여러 메시지 빠르게 보낼 때** — 마지막 인입의 핸들러가 모두 묶어 1회만 답장 (coalesce)
- **부분 가능일 답변** ("수요일은 못해요") → AI 임의 답변 안 함, 즉시 pause + 슬랙
- **연습용 ↔ 라이브 시작 멘트는 한 곳(DB)에서 관리** — 연습에서 다듬으면 라이브에도 그대로 반영
- **지점 활성 토글 OFF는 /apply에만 영향** — admin 모든 화면은 그대로 표시
- **확정 단계는 자동 발송 X** — 매니저 수동 진입은 stage·status만 갱신. 첫 출근 룰은 별도 발송
- **만남장소 자동 발송 없음** — 아이디·차량번호 둘 다 수신되면 슬랙 '준비 완료' 알림 → 매니저가 직접
