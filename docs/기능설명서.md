# 1. 지원 폼 (`/apply`)

## 🎯 기능 목적
**배송원 지원자가 정보를 입력하고 접수하는 창구.**
메타 광고, 알바몬, 카카오 오픈채팅 등 어디서 들어오든 결국 이 폼 하나로 모입니다. 시트 12개를 따로 보던 과거를 끝내고, 모든 지원자를 한 곳에 모으는 입구 역할을 합니다.

## 📋 기능 상세 설명

### 어떻게 들어오나
광고나 홍보 링크에 `?source=meta&branch=강북미아` 같은 식으로 출처와 지점을 미리 붙여 보내면, 지원자가 폼을 열었을 때 1지망 지점이 자동으로 채워져 있습니다. 매니저가 지점별 광고 효과를 추적할 때 이 정보가 그대로 쌓입니다.

### 무엇을 입력받나
- **기본 정보**: 이름, 생년월일 6자리, 휴대폰 번호, 거주지(다음 우편번호 검색 연동)
- **운전 관련**: 자기 차량 여부, 운전면허 종류, 차종
- **희망 사항**: 1지망/2지망 지점, 희망 시간대(평일오전·평일오후·주말오전·주말오후 중복 선택 가능)
- **자기소개·경력·시작 가능일**
- **본인 명의 업무 가능 여부** (정산을 본인 명의로 받을 수 있는지)
- **마케팅 수신 동의** (체크박스, 기본 켜짐) — 추후 추가 모집 SMS 발송 대상 결정에 사용

### 자동 필터 (서류심사 통과 여부)
다음 **3가지 모두**를 만족해야 자동으로 "서류심사" 상태가 됩니다:
1. 자기 차량 **있음**
2. 운전면허 **1종 보통 / 2종 보통 / 1종 대형** 중 하나
3. 본인 명의 업무 **문제 없음**

하나라도 어긋나면 **부적합** 상태로 들어가 매니저 검토 대상에서 제외됩니다.

### 접수 직후 자동으로 일어나는 일
1. **알림톡(또는 SMS) 자동 발송** — "지원이 정상 접수되었습니다, 1~2영업일 내 연락드립니다"
2. **카카오톡 채널 추가 안내 페이지** 노출 — 지원자가 채널을 추가하면 향후 친구톡 발송 대상이 됨
3. **거주지 주소 → 위경도 변환** — 나중에 "픽업지 가까운 후보 추천" 기능에서 사용
4. **구글 시트에도 동기화** — Supabase가 메인이지만 시트는 매니저용 백업/뷰
5. **중복 전화번호 감지** — 이미 지원한 적 있는 사람이면 "중복" 표시가 자동으로 붙음 (저장은 그대로 진행)

## ⚠️ 기타 주의 사항
- **모든 채널의 지원자는 반드시 이 폼을 거쳐야 합니다.** 외부에서 받은 명단은 `지원자 목록`이 아니라 추후 도입될 `수동 입력 폼`으로 들어가야 합니다(현재 미구현).
- 알림톡 템플릿이 심사 통과 전에는 SMS로 자동 폴백됩니다 — 운영에 영향 없습니다.
- 슬랙 알림은 현재 **꺼져 있습니다**. 신규 지원자 즉시 알림이 필요하면 운영팀에 요청하세요.
- 마케팅 수신을 동의 안 한 지원자에게는 광고성 메시지를 보내면 안 됩니다(법적 이슈). 추천 탭의 일괄 발송이 자동 제외하는지 확인 필요.

# 2. 관리자 대시보드 (`/admin`)

좌측 사이드바에 8개 탭이 있습니다. 각 탭의 용도는 아래.

---

## 2-1. 대시보드 탭

### 🎯 기능 목적
**오늘 채용이 어디까지 와 있는지 한눈에 파악.**
출근하자마자 여기를 보면 "어제 몇 명 들어왔고, 스크리닝 밀린 게 몇 명이고, 어느 지점이 부족한지" 5초 안에 알 수 있습니다.

### 📋 기능 상세 설명

상단에 6개 숫자 카드:
- **전체 지원자** — 누적 인원
- **오늘 지원** — 오늘 새로 들어온 사람
- **필터 통과** — 자동 필터를 통과해 검토 대기 중인 사람
- **스크리닝 대기** — 아직 전화 안 한 사람 (밀리면 노란 강조)
- **스크리닝 완료** — 통화 끝나고 가이드 받은 사람
- **확정** — 슬롯에 배치된 사람

아래로는 **지점별 표**가 있어서 지점마다:
- 전체 지원자 수
- 필터 통과 수
- 스크리닝 대기 수
- 통과율

이 비교됩니다. "어느 지점이 모집이 잘 되고 있나, 어디가 마른 곳인가" 판단에 사용.

### ⚠️ 기타 주의 사항
- 숫자는 **실시간 갱신**됩니다 (다른 매니저가 처리하면 자동 반영, 새로고침 불필요).
- "오늘 지원"은 한국 시간 기준 자정~현재까지.
- 통과율이 낮은 지점은 광고 문구나 지점 매력도 점검 신호로 보면 됩니다.

---

## 2-2. 지원자 목록 탭

### 🎯 기능 목적
**모든 지원자를 검색·필터로 찾고, 상태를 바꾸는 메인 작업 화면.**
"이 사람 어떻게 됐지?" 하면 99% 여기서 답이 나옵니다.

### 📋 기능 상세 설명

#### 위쪽 필터·검색
- 지점 드롭다운 / 상태 드롭다운 / 이름·전화번호 검색
- 결과 카운트가 제목 옆에 표시됨

#### 가운데 표 (지원자 행)
각 행에는 이름·연락처·지점·차량·면허·시작가능일·상태·채널·지원일이 표시됩니다.

- **이름 클릭** → 그 사람과의 SMS 대화창이 바로 열림 (배송원 컨택 탭과 동일)
- **행 클릭** → 아래쪽에 상세 패널이 펼쳐짐
- **`1차 스크리닝 완료` 버튼**이 보이는 행 → 클릭하면 가이드 알림톡(템플릿 ⑥)이 자동 발송되고 상태가 "스크리닝 완료"로 바뀝니다
  - 단, **필터 통과(서류심사 상태)인 사람에게만 보입니다.**
- **"중복" 칩**이 이름 옆에 붙은 사람 → 같은 전화번호로 이전에도 지원한 이력이 있는 사람

#### 아래쪽 상세 패널
지원자 한 명을 골라 펼치면 두 영역이 나옵니다:

**(1) 편집 가능 영역** — 드롭다운/날짜로 바꾸고 **반드시 [저장] 버튼**을 눌러야 반영
- 진행 상태 (서류심사 / 스크리닝 완료 / 확정 / 이탈 / 부적합)
- 확정 슬롯 (4슬롯 중 하나)
- 확정 지점
- 현재 근무 지점 (이 사람이 지금 어느 지점에서 일하고 있는가)
- 시작일
- **이탈 사유** (상태를 "이탈"로 바꿨을 때만 입력칸이 나타남)

> 상태를 "이탈"로 바꾸면 자동으로 현재 근무 지점이 비워지고 이탈 시각이 기록됩니다.

**(2) 읽기 전용** — 거주지·차종·희망 지점·희망 시간·자기소개·경력 등

### ⚠️ 기타 주의 사항
- **저장 안 누르고 X로 닫으면 변경사항이 날아갑니다.** "변경됨" 칩이 뜨면 저장 또는 취소를 명시적으로 누르세요.
- 확정 슬롯/지점을 임의로 지정해도 정원(2명) 검사를 안 합니다 — 매니저가 확정 슬롯 매트릭스로 확인하며 배치하세요.
- 현재 페이지네이션이 없어서 누적 5,000명을 넘으면 느려질 수 있습니다 (현재는 문제없음).
- 현재 운영하는 상태는 5종(서류심사·스크리닝 완료·확정·이탈·부적합)입니다. 사양서에 있는 "온보딩/현장투입/대기" 등은 코드에는 아직 없습니다.

## 2-3. 희망 슬롯 탭

### 🎯 기능 목적
**"어느 지점·시간대에 모집을 더 해야 하나?" 를 결정하는 지도.**
지원자가 **희망한** 지점×시간대 분포를 표로 보여줍니다. 1·2지망 모두 카운트됩니다.

### 📋 기능 상세 설명

행 = 지점, 열 = 4슬롯(평일오전·평일오후·주말오전·주말오후) 매트릭스.

- 셀의 숫자 = 그 지점·슬롯을 희망한 활성 지원자(서류심사·스크리닝 완료·확정 상태인 사람) 수
- **색깔로 풀 두께 표시**:
  - 회색(0명) — 위험. 모집 시급
  - 연 노랑(1~2명) — 부족
  - 진한 노랑(3명+) — 여유 있음
- 우측 끝에 **지점 합계**, 아래 끝에 **슬롯 합계** 행
- **셀을 클릭하면** 해당 조건에 맞는 지원자 목록이 아래에 펼쳐집니다 → 이름을 클릭하면 지원자 목록 탭으로 점프해 상세 정보를 볼 수 있습니다

### 어디에 쓰나
- 광고 예산 배분 — 회색·연노랑 셀이 많은 지점·슬롯에 광고 집중
- 슬롯별 채용 우선순위 결정
- "이 지점 평일오전이 1명뿐인데 결원 났네" → 즉시 신규 광고 트리거

### ⚠️ 기타 주의 사항
- **이미 이탈하거나 부적합인 사람은 카운트되지 않습니다.** 살아있는 풀만 보여줍니다.
- 1지망과 2지망을 동등하게 카운트하므로, 같은 사람이 여러 지점 셀에 동시에 들어갑니다(의도된 동작).
- 합계가 전체 지원자 수보다 클 수 있는 이유: 같은 사람이 여러 슬롯·여러 지점을 희망했기 때문.

## 2-4. 확정 슬롯 탭

### 🎯 기능 목적
**"지점별 정원이 다 찼나? 어디가 비었나?" 를 확인하는 화면.**
실제로 배치 확정된 인원만 카운트해서 정원 충족도를 색으로 보여줍니다.

### 📋 기능 상세 설명

행 = 지점, 열 = 4슬롯 매트릭스.
각 셀에 `n/2` 표시 (정원 2명 기준).

색상 의미:
- 🟢 **초록 (2명 이상)** — 정원 충족
- 🟡 **노랑 (1명)** — 절반만 채워짐
- ⚪ **회색 (0명)** — 빈 슬롯

**셀 클릭** → 그 지점·슬롯에 배치된 사람이 누구누구인지 표로 펼쳐짐 (이름·연락처·시작일·현재 근무지). 이름 클릭 시 지원자 목록 탭으로 점프.

### 어디에 쓰나
- 회색 셀 = 빈 자리 → 희망 슬롯 탭으로 가서 후보 찾고 확정 처리
- 노랑 셀 = 1명만 있어서 결원 위험 → 추가 1명 또는 대기자 확보
- 정원 초과(3명 이상)도 그대로 표시되므로 이중 배정 발견용으로도 사용

### ⚠️ 기타 주의 사항
- **현재는 "확정" 상태만 카운트합니다.** 사양서에 있던 "대기 1명" 슬롯은 아직 미구현 — 슬롯당 보여주는 정원은 항상 2명 기준.
- 정원 2명은 코드에 박혀 있습니다. 지점·슬롯별로 정원을 다르게 두려면 개발 요청이 필요합니다.
- 출근 전날 확인(템플릿 ⑤) 자동화는 아직 없습니다 — 시작일이 임박한 사람은 매니저가 직접 챙겨야 합니다.

## 2-5. 배송원 추천 탭

### 🎯 기능 목적
**"이 공고에 누구한테 연락할까?" 를 AI가 추천하고, 그대로 일괄 SMS로 보낼 수 있는 화면.**
과거에 지원했지만 아직 일하지 않는 사람들 + 레거시 명단 중에서 픽업지에 가깝고 차량 조건 맞는 후보를 점수로 정렬해 줍니다.

### 📋 기능 상세 설명

#### 1단계 — 공고 작성
- **자동 생성**: "강북미아 토일 장보기 자차, 시급 1.5~2만, 픽업 도봉로 34" 같이 대충 입력하면 Claude가 정형 공고로 다듬어 줍니다. 빠진 항목은 `[?]`로 표시되니 직접 채우세요.
- 또는 직접 공고 텍스트 작성

#### 2단계 — 옵션 선택
- 차량 필요 / 차량 불필요 (필수)
- (선택) 상차지 주소 직접 입력 — Claude가 공고에서 자동으로 뽑지만 빗나갈 때 수동 지정

#### 3단계 — 추천 받기
- `[추천 받기]` 클릭 → Claude가 공고에서 픽업 주소를 추출 → 거주지 가까운 순 + 차량 조건 + 최신성 점수로 상위 10명 표시
- 점수 만점 100점 = 거리 70 + 차량 20 + 최신성 10
- 부족하면 `10명 더 받기`로 최대 50명까지 확장
- 표에 순위·이름·나이·출처(신규/레거시)·거리(km)·차량·시군구·점수가 표시됨

#### 4단계 — 발송
- 체크박스로 받을 사람 선택 (전체선택/해제 버튼)
- 하단에 **발송 미리보기**(본문 + 받는 사람 칩) 표시
- `[N명에게 발송]` → 공고 내용이 그대로 SMS로 일괄 발송

### 어디에 쓰나
- 갑자기 결원이 생긴 슬롯에 채울 사람을 즉석에서 찾아 메시지 일괄 발송
- 신규 지점 오픈 시 인근 거주 후보 풀 동원
- 레거시(과거 지원자) 재활용

### ⚠️ 기타 주의 사항
- **한 번 발송된 SMS는 회수 불가합니다.** 미리보기 꼭 확인.
- 점수 가중치(거리 70/차량 20/최신성 10)는 코드에 박혀 있습니다. 운영하면서 비율 조정이 필요하면 개발 요청.
- **희망 시간대(work_hours)와 공고 시간대 매칭은 안 합니다** — 거리·차량·최신성만 봅니다. 평일오전 공고에 주말만 희망한 사람도 후보에 들어올 수 있으니 매니저가 검토 후 발송.
- 마케팅 미동의자가 자동 제외되는지 운영팀이 확인 필요 (법적 이슈).
- 친구톡(카카오 채널 친구) 발송 분기는 아직 미구현 — 모두 SMS로 나갑니다.

## 2-6. 배송원 컨택 탭

### 🎯 기능 목적
**지원자와의 SMS 양방향 대화를 카톡처럼 관리하는 화면.**
전용 폰을 직접 들고 보지 않고, 대시보드 안에서 답장하고 받습니다. AI가 답변 초안까지 자동으로 만들어 줍니다.

### 📋 기능 상세 설명

#### 컨택 리스트
- **안 읽은 메시지가 있는 사람이 항상 맨 위**에 옵니다 (빨간 배지로 개수 표시)
- 그 다음 최근 문자 순 → 문자 없는 사람은 지원일 순
- 카드 클릭 → 카톡 스타일 대화창이 오버레이로 열림

#### 대화창
- 상대 메시지(왼쪽 회색) / 우리 메시지(오른쪽 색) 말풍선
- 발송 시각, 보낸 사람(시스템 자동 / 매니저 ID) 표시
- 입력창에서 Enter = 발송, Shift+Enter = 줄바꿈

#### 🤖 AI 답변 초안 (핵심)
지원자가 답장을 보내면 자동으로:
1. Claude가 대화 맥락 + 공고 + 지원자 정보를 보고 답변 초안을 만듭니다
2. 입력창 위에 노란 박스로 초안이 뜹니다 (판단 근거도 함께)

매니저가 선택할 수 있는 행동 3가지:
- **그대로 보내기** — 초안을 그대로 전송
- **수정해서 보내기** — 초안이 입력창에 들어와 편집 후 전송
- **무시** — 박스만 닫음

만약 AI가 답변하기엔 정보가 부족하다고 판단하면 **⚠️ AI 응대 불가** 상태로 바뀌고, "이 정보가 모자랍니다" 안내 + 슬랙 알림이 가서 매니저가 직접 답해야 합니다.

### 어디에 쓰나
- 지원 직후 문의("저 차 없는데 가능해요?") 즉시 응대
- 스크리닝 일정 조율
- 출근 직전 안내·확인
- 매니저 1명이 200명을 동시 컨택 가능하게 만드는 도구

### ⚠️ 기타 주의 사항
- **초안은 어디까지나 제안.** 그대로 보내기 전에 한 번 읽으세요. 특히 가격·시간·조건은 잘못 나갈 수 있습니다.
- 카카오 1:1 채팅 답장은 카카오 정책상 API로 안 됩니다 — 무조건 SMS 양방향만 가능합니다.
- 첨부 파일/이미지는 미지원.
- 전용 폰(SMS Gateway)이 5분 이상 신호 끊기면 수신이 멈출 수 있습니다 — heartbeat 모니터링 별도.
- 안 읽음 카운트는 채팅창을 열면 자동으로 0이 됩니다.

## 2-7. 지점 관리 탭

### 🎯 기능 목적
**`/apply` 폼의 지점 드롭다운, 모든 통계·필터의 지점 목록을 관리.**
신규 지점 오픈, 휴업, 폐점, 이름 변경 시 사용합니다.

### 📋 기능 상세 설명

- **지점 추가**: 상단 입력창에 이름 입력 → `+ 지점 추가`
- **순서 변경**: 행을 드래그(좌측 `⋮⋮` 핸들)해서 위아래로 이동 — `/apply` 폼 드롭다운 순서가 이대로 노출됨
- **이름 변경**: 표 안의 이름 input을 직접 편집
- **활성 토글**: 체크박스 끄면 지원 폼 드롭다운에서 사라지고 통계에서도 제외됨 (데이터는 유지)
- **삭제**: 우측 삭제 버튼

> 모든 변경은 **반드시 [변경사항 저장] 버튼**을 눌러야 적용됩니다.

각 행 우측에 그 지점을 사용 중인 지원자 수("지원자 12명")가 표시됩니다.

### 어디에 쓰나
- 신규 지점 오픈 (예: 송파잠실)
- 임시 휴업 시 활성만 끄기 (지점 데이터 보존)
- 지원 폼 드롭다운 순서를 광고 우선순위에 맞게 조정

### ⚠️ 기타 주의 사항
- **지점 이름을 바꾸면 기존 지원자의 지점 정보가 자동으로 안 따라옵니다.**
  예: "강북미아"를 "강북"으로 바꾸면 기존 지원자들은 여전히 "강북미아"로 표시됩니다 → 통계에서 제대로 안 잡힐 수 있습니다.
  → **이름 변경은 가급적 피하고, 폐점은 비활성으로 처리하세요.**
- 사용 중인 지점을 삭제하면 기존 지원자가 어떻게 표시되는지 운영팀 확인 필요.
- 사양서에 적힌 12지점은 참고용이고, **실제 운영은 이 탭의 목록 기준**입니다.

## 2-8. 구인 에이전트 탭

### 🎯 기능 목적
**"AI가 구직자 문의에 어떻게 답하는지" 미리 시뮬레이션해 보는 테스트장.**
실제 SMS 발송도 안 되고, DB 저장도 안 됩니다 — 톤·답변 정확도를 매니저가 점검할 때 사용.

### 📋 기능 상세 설명

#### 좌측: 구인공고
- 구직자가 보고 문의온다고 가정할 공고 본문을 붙여넣음
- 누적 대화 턴 수 표시
- `🔄 대화 초기화` — 새 시나리오로 다시 시작

#### 우측: 가상 채팅창
- 입력창에 "구직자가 보낼 법한 메시지"를 입력 → Enter
- AI가 답변을 생성해서 대화에 추가
- 각 턴은 **편집 가능**합니다 — 메시지 내용을 수정하면 다음 호출 시 그 수정본이 맥락으로 들어갑니다
- 턴 우상단 ✕로 그 턴부터 뒤를 통째로 삭제 가능
- AI 답변에는 **판단 근거**(왜 이렇게 답했는지)와 부족한 정보가 함께 표시됨

### 어디에 쓰나
- 새 공고를 만들 때 다양한 구직자 패턴(애매한 질문, 까다로운 조건)으로 미리 응대 시뮬레이션
- AI가 잘못된 답을 하면 → `prompts/conversation-examples.txt` 파일을 수정해 학습 예시 보강 후 배포
- 톤이 너무 딱딱하거나 너무 친근한지 점검

### ⚠️ 기타 주의 사항
- **실제 발송 안 됩니다.** 마음껏 테스트하세요.
- 새로고침하면 대화가 사라집니다 (브라우저 메모리에만 있음).
- 실제 운영(컨택 탭의 자동 초안)은 이 테스트와 달리 **지원자 DB까지 컨텍스트로** 사용합니다 — 테스트에서 잘 답하는 게 운영에서도 잘 답한다는 보장은 없습니다(반대도 마찬가지).
- 프롬프트 변경은 코드 배포가 필요합니다 — 개발팀에 요청.

# 3. 전체 운영 흐름 한눈에 보기

```
[광고/홍보 링크 클릭]
        ↓
   /apply 폼 작성
        ↓
   자동 필터 (차량·면허·본인명의)
        ↓
   ┌─ 통과 → 상태=서류심사 + 알림톡 ① 자동 발송 + 시트 동기화
   └─ 탈락 → 상태=부적합 (이후 자동 처리 없음)
        ↓
   24시간 무응답이면 → 알림톡 ② 리마인더 (매일 1회 자동)
        ↓
   매니저: 지원자 목록에서 전화 스크리닝
        ↓
   매니저: [1차 스크리닝 완료] 클릭
        ↓
   알림톡 ⑥ 가이드 자동 발송 + 상태=스크리닝 완료
        ↓
   매니저: 상세 패널에서 확정 슬롯/지점/시작일 입력 + 상태=확정
   (※ 확정 알림톡 ③은 아직 자동화 X — 컨택 탭에서 수동 발송)
        ↓
   확정 슬롯 매트릭스에 자동 카운트
        ↓
   근무 시작 시 매니저가 [현재 근무 지점] 입력
        ↓
   이탈 시 상태=이탈로 변경 (이탈 시각·근무지 자동 정리)
        ↓
   결원 발생 시 → 추천 탭에서 풀 다시 발굴
```

## 매일/매주 매니저가 할 일 요약

| 빈도 | 작업 |
|--|--|
| **매일** | 대시보드 확인 → 컨택 탭의 안 읽음 처리 → 지원자 목록의 "1차 스크리닝 완료" 가능 행 처리 |
| **수시** | 결원/긴급 모집 시 추천 탭으로 일괄 발송 |
| **주 1회** | 희망 슬롯 매트릭스로 풀 부족 지점·슬롯 점검 → 광고 예산 조정 |
| **주 1회** | 확정 슬롯 매트릭스로 정원 미달 슬롯 채움 |
| **분기 단위** | 지점 관리 탭 정리, 사용 안 하는 지점 비활성 |

## 자주 헷갈리는 포인트

- **"지원자 목록"의 상태 변경은 [저장] 버튼을 눌러야 반영됩니다.** X로 닫으면 날아갑니다.
- **알림톡이 SMS로 나가는 건 정상**입니다 — 템플릿 심사 통과 전에는 폴백 동작.
- **추천 탭의 일괄 발송은 회수 불가** — 미리보기 필수.
- **지점 이름 변경은 위험** — 비활성으로 끄세요.
- **AI 답변 초안은 항상 검수 후 발송** — 가격·시간·조건은 특히.
- **사양서와 실제 운영이 어긋나는 부분이 있습니다** (상태 종류, 대기 슬롯 등) — 헷갈리면 화면이 정답.
